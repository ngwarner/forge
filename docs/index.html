How it works

Three steps. No code.

Forge separates understanding from building — the step where AI usually goes wrong.

Step 1
Describe your need
Say what's hard, what you need, or what you want to build. Plain English. No technical terms required.
Step 2
Review the spec
Forge shows you what it understood before writing a single line of code. You approve, correct, or redirect — then it builds.
Step 3
Get working software
A complete, immediately usable app or script. Not a prototype. Not starter code. The finished thing.
Always
Verified against the spec
The examples you approved run automatically against the output. If they fail, Forge regenerates — not you.
Why it's safe

Built for humans, not developers.

Forge is designed around one principle: you should always understand what's being built before it's built.

📋
Spec before code
AI shows its interpretation in plain English before generating anything. You approve it. Misunderstandings get caught here — not after.
🔒
Declared effects only
Every side effect (network calls, file access, location) is declared in the spec. An implementation that does more violates the contract.
🔑
Your key, your data
Forge never stores your API key on a server. It lives in your browser and is sent directly to the AI. Nothing is collected.
✓
Auto-verified
The examples in your spec become automatic tests. If the built app fails them, Forge knows — and rebuilds before you see it.
